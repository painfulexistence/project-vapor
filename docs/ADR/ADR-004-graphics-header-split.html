ADR-004: Graphics Header Split

2026-04-29 Adopted

Context

Vapor/include/Vapor/graphics.hpp was a 543 LOC god header containing GPU handles, 2D batch structures, GPU layout structs, CPU-side mesh/material data, and visual effect structs — 40+ types in one file.

Any change to any struct triggered recompilation across all 14 files that included the header, regardless of which section actually changed.

Decision

Split graphics.hpp into five semantic sub-headers, keeping graphics.hpp as a zero-breakage umbrella include.

Sub-headerContents
graphics_handles.hppGPUHandle<Tag> template + all handle typedefs
graphics_batch2d.hppBlendMode, Batch2DVertex, Batch2DUniforms, Batch2DStats
graphics_gpu_structs.hppMaterialData, lights, CameraData, InstanceData, Cluster, LightCullData, IBLCaptureData
graphics_mesh.hppAlphaMode, Image, Material, VertexData, Mesh
graphics_effects.hppWater, Atmosphere, VolumetricFog/Cloud, LightScattering, SunFlare, GPU particles

Additional Decision: Handle Type Safety

The original four GPU handles (PipelineHandle, BufferHandle, TextureHandle, RenderTargetHandle) were structurally identical bare structs. The type system provided no protection against passing a BufferHandle where a TextureHandle was expected.

Replaced with a GPUHandle<Tag> template:

template<typename Tag>
struct GPUHandle {
    Uint32 rid = UINT32_MAX;
    bool valid() const { return rid != UINT32_MAX; }
};
using BufferHandle  = GPUHandle<BufferHandleTag>;
using TextureHandle = GPUHandle<TextureHandleTag>;
// ...

Passing a BufferHandle where a TextureHandle is expected is now a compile error, at zero runtime cost.

Migration

  • All existing #include "graphics.hpp" sites require no changes — the umbrella re-exports everything
  • graphics.cpp switches to #include "graphics_mesh.hpp" (only needs Mesh implementation)
  • New files should include the most specific sub-header needed

Consequences

  • 5 new header files
  • graphics.hpp reduced to 15 lines
  • Handle types are no longer interchangeable in the GPU layer
  • New effect structs have a clear home (graphics_effects.hpp)
